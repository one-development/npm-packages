# Style Props Reference

Many One UI components support _Style Props_, which make it easy to use theme values for CSS properties like _margin_, _color_, _height_, or _width_. Style Props make components more declarative and eliminate the need to write custom style rules.

- [Basics](#basics)
- [Responsive Values](#responsive-values)
- [Prop Categories](#prop-categories)

## Basics

##### Overview

Style Props are a concept borrowed from the [Styled System](https://styled-system.com/getting-started) library. It is _highly_ recommended that you take a look at Styled System's documentation before continuing with this reference.

##### Box

Style props pull values from the [theme](/?path=/story/theme-reference--page). Many components support Style Props, but [Box](/?path=/docs/components-container--basic) is the primitive building block for interacting with them.

```javascript
<Box
  color='onSurface.primary' // theme.color.onSurface.primary
  margin='md' // theme.space.md
  height='100%' // uses 100% height as custom value
  width='containedWidth' // theme.sizes.containerWidth
>
  This is a box
</Box>
```

##### Component Docs

When a component supports a _Style Prop_, it will be labeled accordingly. The propType will support `number | string | array | object` and its description will include a label that says `responsive prop`. A full list of style props is documented in [Box](/?path=/docs/components-container--basic) and other components that support them.

## Responsive Values

When working with responsive layouts, it's often useful to adjust styles along a singular dimension, such as font-size, margin, padding, or width. Instead of manually managing media queries for each breakpoint and adding nested style objects throughout your codebase, Style Props support a shorthand syntax for adding responsive styles with a mobile first approach. While this syntax can seem odd at first, it can become a powerful way to manage responsive typography and layouts. All style props support arrays as values for mobile-first responsive styles. Here's an example:

```javascript
// Or you can pass in an object, using breakpoint aliases as keys.
// Note that the "default" key is any key that's not a breakpoint alias and can be named to what you prefer.
<Box
  marginX={{ default: 'md', xs: 'lg', md: 'auto' }}
>
  This is a box
</Box>

// Or you can pass in an array
<Box
  marginX={[
    'md', // 16px below the smallest breakpoint
    'lg', // 24px from the next breakpoint and up
    null, // Skip next breakpoint
    'auto', // auto from the next breakpoint and up
  ]}
>
  This is a box
</Box>
```

The example would generate something like the following CSS:

```css
.Box-hash {
  marginLeft: 16px
  marginRight: 16px
}

@media screen and (min-width: 360px) {
  .Box-hash {
    marginLeft: 24px
    marginRight: 24px
  }
}

@media screen and (min-width: 960px) {
  .Box-hash {
    marginLeft: auto
    marginRight: auto
  }
}
```

If you need additional info on _responsive styles_, check out this [guide](https://styled-system.com/responsive-styles) from Styled System.

### Space

```js
import { space } from 'styled-system'
// or `import space from '@styled-system/space'`
;<Box m={2}>Tomato</Box>
```

| Prop                  | CSS Property                       | Theme Field |
| --------------------- | ---------------------------------- | ----------- |
| `m`, `margin`         | `margin`                           | `space`     |
| `mt`, `marginTop`     | `margin-top`                       | `space`     |
| `mr`, `marginRight`   | `margin-right`                     | `space`     |
| `mb`, `marginBottom`  | `margin-bottom`                    | `space`     |
| `ml`, `marginLeft`    | `margin-left`                      | `space`     |
| `mx`, `marginX`       | `margin-left` and `margin-right`   | `space`     |
| `my`, `marginY`       | `margin-top` and `margin-bottom`   | `space`     |
| `p`, `padding`        | `padding`                          | `space`     |
| `pt`, `paddingTop`    | `padding-top`                      | `space`     |
| `pr`, `paddingRight`  | `padding-right`                    | `space`     |
| `pb`, `paddingBottom` | `padding-bottom`                   | `space`     |
| `pl`, `paddingLeft`   | `padding-left`                     | `space`     |
| `px`, `paddingX`      | `padding-left` and `padding-right` | `space`     |
| `py`, `paddingY`      | `padding-top` and `padding-bottom` | `space`     |

Styled System provides subsets of `space` category: `margin` and `padding`.
